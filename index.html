Model Selection

Choose models based on your hardware. Larger models are more accurate but require more memory and take longer to download. Changes take effect after clicking Reload Models.

Local Security

Install this app from a trusted HTTPS or localhost origin. After the selected models finish loading once, the app shell and fetched model assets are available from the browser cache for local offline use. Keep patient identifiers out of persistent settings.

Medical Terminology

One term per line. During transcript cleanup, the LLM will use this list to correct any word that appears phonetically garbled or oddly spelled in a medical context — medications, diagnoses, procedures, anatomical terms, brand names, or anything else that Whisper commonly mishears. Add or remove terms based on what your practice uses. No need to predict how a word was misheard — the LLM fuzzy-matches against this list automatically.

Shorthand Macros

Add shortcuts that expand automatically as you type/edit or during note generation. Format as .shortcut: expansion (one per line). Starting shortcuts with a dot (like .aom: acute otitis media) makes them easy to trigger without expanding regular words.

System Prompts

These prompts are sent to the LLM on every request. The medical terminology list above is appended to the cleanup prompt automatically — no need to add it here.

Extra Pipeline Steps

Optional prompts run after the main note is generated. Each enabled step gets its own output box with a separate copy button.

Templates

SmartChart templates match typed keywords live, assemble in priority order, and copy automatically after the configured delay.

Boilerplate Entries

Each entry has a key (used in [BOILERPLATE:KEY] tags), optional trigger phrases that tell the LLM when to emit the tag, and the boilerplate text injected into the output.
